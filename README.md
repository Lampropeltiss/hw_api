# Курсовая работа «Резервное копирование»

### Входные данные:
- токены VK и Яндекс.Диска (вводятся в config.ini, шаблон приложен);
- ID пользователя VK;
- альбом (профиль или стена);
- желаемое количество изображений для загрузки на Яндекс.Диск;
- выбор сохранения на пк.

### Выходные данные:
- json-файл с информацией по изображениям, загружаемым на Яндекс.Диск;
- изменения на Яндекс.Диске (созданная папка и загруженные туда изображения);
- прогресс-бары для отслеживания процесса программы (исп. tqdm, первый опыт).

### Примечания:
- форматы размеров, используемые VK, мне показались неудобными. Было решено использовать данные о ширине/высоте изображений;
- изображения сортируются по площади в пикселях, затем отбирается нужное количество самых больших из них;
- в json-файл размер записывается в формате [width]x[height];
- загрузка изображений на Яндекс.Диск реализована через ссылки, полученные из VK, поэтому загрузка всех или выбранных изображений на пк опциональна;
- реализовано получение имени пользователя для названия папки на диске;
- если количество лайков одинаково, то формат названия изображения следующий: [кол-во лайков] [dd-mm-YYYY].jpg.
